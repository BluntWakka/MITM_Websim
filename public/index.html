Welcome to the Man-In-The-Middle Testing Application! Below is some information on each of the entries and their meaning.
Plaintext/Password: The text passed in and the desired information that an attacker would be attempting to decrypt.
Encrypted Text: The resultant text based on a combination of the given plaintext, the key entered, and the encryption scheme used.
Key: A key is a string of information that is used in an encryption scheme as the catalyst to base the ciphertext on. The same key will not produce the same result in a given scheme even if the entered information to encrypt is the same. If using a good encryption method, other charcters are typically intermingled and appened to keys in via salting and IVs. If no key is enetered and the encryption scheme requires one, a key will be generated and the text "No key used" will still be displayed.
Method of Encyrption: An encryption scheme is something that takes plaintext and transforms it into unusable text unless one posesses a key, thereby securing it against prying eyes and common attacks like credential stuffing.
Attacker type: Some attackers may use intricate networks of computers in order to efficiently split workloads and increase the effective computational power attempting to crack a given cipher. General metrics of how many computers are used in such schemes are provided.
GPU Type: Attackers and bad actors will utilize differnt GPUs (Graphics Processing Units) in order to check more possible keys per second. Different types of GPUs have widley varying strength, and some encryption methods are built specifically to combat mass hashing.
Entropy Bits: Entropy bits are a measure of how many attempts are needed to crack a given password. To get an estimate of the number of hashes necessary to crack a given ciphertext from this figure (excluding salting or IVs), simply calculate 2^(Entropy bits).
Scheme Hardness Factor: Certain encryption schemes have varying levels of difficulty to compute a single hash. This factor uses the MD5 scheme as a base point for the value one.
Time to Crack: The estimated time it would take given all previous conditions to crack a ciphertext (information based on real world efficient cracking techniques from Hashcat).
